## 一、測試前準備

請在正式測試前確認以下項目：

- [ ] 後台系統檢查無紅色 ❌ 錯誤
  - 登入後台 → 切換「系統檢查」tab → 點「重新檢查」
  - 若有 ❌，先處理完再進行測試
- [ ] 菜單品項已完整輸入，分類與名稱正確
- [ ] 店名、電話、地址、Logo 已在「店家設定」填妥並儲存
- [ ] QR Code 可正常掃描（以手機實際掃一次，確認開啟的是本店點餐頁）
- [ ] owner 帳號已可登入後台
- [ ] 至少一個 staff 帳號已建立並可登入
- [ ] 前台點餐頁網址已確認：`https://ririchu-14b81.web.app/?store=ririchu`

> 若任一項未完成，請先處理後再進行測試。

## 二、30 分鐘快速驗收

不需要真實顧客，由測試人員自行操作，確認核心流程通暢。

### 前台點餐（10 分鐘）

- [ ] **內用訂單**
  - 手機開啟前台 → 選品項 → 切換「內用」→ 填桌號 → 送出
  - 確認出現「訂單 #xxxxxx 已送出」成功提示

- [ ] **外帶訂單**
  - 切換「外帶」→ 填姓名、電話、選取餐時間 → 送出
  - 確認出現送出成功提示

- [ ] **規格品項**
  - 若菜單有規格品項（例如：大/小杯），點選後確認規格選擇器出現
  - 選完規格後加入訂單，確認品項名稱包含規格

- [ ] **法律連結出現**
  - 點「查看訂單」開啟訂單明細 drawer
  - 確認送出按鈕上方有「隱私權政策與服務條款」連結
  - 點連結確認可開啟法律頁面

- [ ] **個資聯絡方式顯示正確**
  - 開啟 `/legal.html?store=ririchu` 確認「聯絡方式」區塊顯示 Email、電話或地址
  - 若尚未填入 contactEmail，確認顯示電話或預設文字

### 後台收單（10 分鐘）

- [ ] **staff 帳號收單**
  - 以 staff 帳號登入後台
  - 確認剛才前台送出的訂單出現在收單頁
  - 點「✓ 完成出餐」，確認訂單從畫面消失

- [ ] **歷史訂單查詢**
  - 以 manager 或 owner 帳號切換到「訂單明細」
  - 確認剛才完成的訂單出現在清單中

- [ ] **CSV 匯出**
  - 在訂單明細頁面按「匯出 CSV」
  - 確認檔案下載，用 Excel 開啟後中文顯示正常，訂單筆數正確

- [ ] **訂單列印**
  - 收單頁有訂單時，點訂單卡片右下角「列印」
  - 確認收據頁面開啟，顯示店名、品項、金額
  - 確認「完成出餐」按鈕仍可正常點按（列印不影響出餐）

### 新功能確認（10 分鐘）

- [ ] **暫停接單**
  - 以 manager 或 owner 登入 → 收單 tab → 點「暫停接單」
  - 前台確認出現橘色暫停橫幅，送出按鈕無法點按
  - 回後台點「恢復接單」，前台橫幅消失

- [ ] **缺貨管理**
  - 收單 tab → 點「缺貨管理」→「展開」
  - 點任一品項「缺貨」→ 前台確認該品項消失
  - 點「供應中」→ 前台確認品項恢復

- [ ] **QR Code 列印**
  - owner 帳號 → QR Code tab → 點「列印 QR Code」
  - 確認 QR Code 名片可列印，名片上店名、地址、電話正確

### 通過標準

以上所有項目均確認後，系統可進入正式使用。
若有任何項目未通過，記錄在下方「問題記錄」後再決定是否正式上線。

## 三、1 天真實營業測試

在真實顧客與尖峰流量下找出設計盲點。建議挑選一個平日或週末的半天（2–4 小時）進行。

### 測試前額外準備

- [ ] 所有需要上班的員工帳號都已建立並測試過可以登入
- [ ] 每個員工裝置登入後台，設定螢幕不自動休眠
- [ ] 準備備用方案（紙本點餐單），以防網路或裝置問題

### 營業中觀察重點

請記錄以下情況是否發生：

| 觀察項目 | 是否有問題 | 備註 |
|---------|----------|------|
| 顧客能否順利掃 QR Code 開啟點餐頁 | | |
| 內用顧客填桌號是否清楚（員工能對到桌）| | |
| 外帶取餐時間選項是否夠用（時段足夠） | | |
| 尖峰時段多筆訂單同時出現，員工能否應付 | | |
| 是否有訂單遺失或沒收到 | | |
| 缺貨時，員工是否需要叫 manager 才能切換 | | |
| 網路不穩時（4G 切 WiFi），訂單是否正常 | | |
| 員工裝置（手機/平板型號）有無顯示問題 | | |

### 收攤後確認

- [ ] 匯出當天 CSV，核對筆數與實際訂單一致
- [ ] 確認沒有「送出成功但後台沒收到」的訂單
- [ ] 向員工詢問：「哪個步驟最卡、最容易搞錯？」
- [ ] 向老闆詢問：「有沒有很想要但沒有的功能？」
