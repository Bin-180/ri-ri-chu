# 備份與匯出 SOP

本文件說明如何定期保留訂單資料，目標讀者為非工程人員（老闆、店長）。

> **重要**：本系統目前以 **CSV 匯出**作為主要備份方式。CSV 適合查詢對帳與財務保存，但不支援一鍵還原整個系統；系統本身的即時資料仍存放在 Firebase，不受此備份操作影響。

---

## 一、備份目的

| 用途 | 說明 |
|------|------|
| 對帳 / 財務申報 | 查詢特定日期的訂單金額、品項數量 |
| 糾紛處理 | 顧客或外送平台有爭議時，可調閱歷史訂單佐證 |
| 稅務保存 | 依法一般建議保存 5–7 年 |
| 資料備援 | 萬一 Firebase 帳號異常，仍有本機副本可查閱 |

---

## 二、備份頻率建議

| 頻率 | 建議操作 | 說明 |
|------|---------|------|
| **每日** | 當天營業結束後匯出當日訂單 | 適合高流量店家（日均訂單 30 筆以上） |
| **每週** | 每週一匯出上週訂單 | 適合中小型店家 |
| **每月**（最低建議） | 每月 1 日匯出上個月訂單 | 最低頻率，資料量較大，請確保硬碟空間足夠 |

> 建議至少每月執行一次，並保存最近 12 個月的 CSV 檔案。

---

## 三、CSV 匯出流程

### 所需權限

**manager** 或 **owner** 皆可執行匯出；staff 無此功能。

### 步驟

**Step 1 — 登入後台**

開啟後台網址（例如：`https://ririchu-14b81.web.app/admin.html`），以 manager 或 owner 帳號登入。

**Step 2 — 切換到「訂單明細」tab**

點擊頂部導覽列的「**訂單明細**」。

**Step 3 — 選擇日期範圍**

在頁面上方選擇要匯出的日期區間：

- 單日：選擇同一天的開始與結束日期
- 整月：例如 2026-04-01 至 2026-04-30

**Step 4 — 點擊「匯出 CSV」**

點擊「**匯出 CSV**」按鈕，瀏覽器會自動下載一個 `.csv` 檔案。

**Step 5 — 確認檔案可開啟**

用 Excel 或 Numbers 開啟下載的 CSV，確認：

- 欄位標題正確（訂單編號、時間、品項、金額等）
- 筆數與當日/當月預期訂單量相符
- 無亂碼（若出現亂碼，請以「UTF-8」編碼重新開啟）

---

## 四、檔案命名與存放建議

建議建立以下本機資料夾結構，方便日後查閱：

```
備份/
└── ririchu/
    ├── 每日/
    │   ├── 2026-05-24.csv
    │   ├── 2026-05-25.csv
    │   └── ...
    └── 每月/
        ├── 2026-04.csv
        ├── 2026-05.csv
        └── ...
```

**命名規則**

| 類型 | 格式 | 範例 |
|------|------|------|
| 單日匯出 | `YYYY-MM-DD.csv` | `2026-05-24.csv` |
| 整月匯出 | `YYYY-MM.csv` | `2026-05.csv` |
| 跨日期區間 | `YYYY-MM-DD_to_YYYY-MM-DD.csv` | `2026-05-01_to_2026-05-15.csv` |

> 建議同時備份到外接硬碟或雲端硬碟（Google Drive、iCloud），避免單一裝置損壞導致資料遺失。

---

## 五、每月備份檢查清單

每月月初執行以下確認：

- [ ] 已匯出上個月完整訂單 CSV
- [ ] 用 Excel / Numbers 開啟確認無亂碼、欄位正確
- [ ] 確認筆數合理（與月底累計訂單量相符）
- [ ] 檔案已存放到本機「每月」資料夾
- [ ] 備份已同步到外接硬碟或雲端硬碟
- [ ] 確認近 12 個月的 CSV 都存在，舊檔未被意外刪除

## 六、復原方式

目前系統**不支援一鍵從 CSV 還原資料庫**，CSV 的用途主要是：

| 用途 | 方法 |
|------|------|
| 查閱歷史訂單 | 直接用 Excel 開啟 CSV 搜尋 |
| 對帳 / 計算金額 | 在 Excel 用加總、篩選等功能計算 |
| 資料遺失補救 | 聯絡開發人員，搭配 Firestore 備份（見下節）進行還原 |

> Firebase 上的即時訂單資料由 Google 負責基礎設施層的可用性。正常使用下，資料不會憑空消失；CSV 備份的主要作用是留存可本地查閱的副本，而非對抗 Firebase 停服。

## 七、進階備份選項（目前未實作）

以下為未來可選的進階方案，**目前均未啟用**，需要額外設定與成本評估。

### Firestore 匯出（完整資料庫備份）

Firebase 提供 Firestore 匯出功能，可將整個資料庫備份到 Google Cloud Storage。

- **需要**：升級至 Blaze（付費）方案、啟用 Cloud Storage bucket
- **操作**：Firebase Console → Firestore → Import/Export → 手動匯出
- **格式**：二進位 Firestore 格式（可用於還原資料庫，非 CSV）
- **費用**：Cloud Storage 儲存費用（小流量極低，約 $0.02/GB/月）

### 自動定期備份（Cloud Scheduler）

可設定排程每日自動觸發 Firestore 匯出。

- **需要**：Cloud Functions + Cloud Scheduler（均需 Blaze 方案）
- **實作時機**：正式大流量上線後，再評估是否需要自動化

### Google Drive 自動上傳

將 CSV 自動上傳至指定 Google Drive 資料夾。

- **需要**：Cloud Functions + Google Drive API 授權
- **替代方案**：手動下載後拖曳到 Google Drive（目前建議做法）

## 八、異常情況處理

### CSV 下載後為空白或筆數異常

1. 確認日期範圍選擇正確（注意月份起訖）
2. 確認當期確實有訂單（「收單」tab 或「統計」tab 確認）
3. 若確認有訂單但 CSV 仍為空，請截圖回報給開發人員

### Excel 開啟後出現亂碼

CSV 使用 UTF-8 編碼。解決方式：

**Excel（Windows）**：
1. 不要直接雙擊開啟
2. 開啟 Excel → 資料 → 從文字/CSV → 選擇檔案 → 編碼選「65001: Unicode (UTF-8)」

**Numbers（Mac）**：直接開啟通常正常；若有亂碼，請用文字編輯器確認儲存時的編碼。

### Firebase 帳號無法登入

若帳號異常導致無法登入後台匯出資料：

1. 請聯絡 owner 帳號協助重設密碼
2. 若 owner 帳號也無法登入，請聯絡開發人員，透過 Firebase Console 直接匯出資料
