# 隱私權政策 / 服務條款草稿

> **重要聲明**：本文件為草稿範本，僅供參考，**非法律意見**。
> 正式上線前，業主應自行確認內容是否符合當地法規（個人資料保護法等），
> 並視需要諮詢法律專業人士後再對外公告。

---

## 一、隱私權政策草稿

### 適用範圍

本隱私權政策適用於透過本點餐系統（以下簡稱「本系統」）進行點餐的顧客，以及使用後台管理功能的店家人員。

---

### 蒐集的個人資料

本系統依據點餐服務需要，蒐集以下資料：

**顧客（前台點餐）**

| 資料項目 | 情境 |
|---------|------|
| 桌號 | 內用點餐時由顧客填寫 |
| 姓名 | 外帶點餐時由顧客填寫 |
| 電話號碼 | 外帶點餐時由顧客填寫，供店家確認取餐 |
| 取餐時間 | 外帶點餐時由顧客選擇 |
| 訂單內容 | 選購品項、數量、規格 |
| 備註 | 顧客自行填寫的特殊需求（例如：少冰、不加辣） |

**店家人員（後台帳號）**

| 資料項目 | 說明 |
|---------|------|
| 電子郵件地址（Email） | 作為登入帳號 |
| 顯示名稱（displayName） | 後台識別用途 |
| 角色（role） | 後台權限分類（owner / manager / staff） |

---

### 使用目的

蒐集的資料僅用於以下目的：

1. **建立訂單**：記錄顧客點選的品項與訂單資訊
2. **出餐與取餐聯絡**：店家依桌號或姓名、電話確認出餐，外帶顧客取餐時核對
3. **訂單查詢與營業統計**：店家查看歷史訂單、統計營業額與銷售數量
4. **帳號權限管理**：管理後台人員的存取權限

本系統不會將個人資料用於廣告投放、第三方行銷或販售。

---

### 資料儲存位置

所有資料儲存於 **Google Firebase（Google Cloud Platform）**，伺服器位於 Google 全球資料中心。

Firebase 的資料安全與隱私政策請參閱：
- https://firebase.google.com/support/privacy
- https://policies.google.com/privacy

---

### 資料保存期間

訂單資料依店家實際營運及稅務申報需求保存。

> 若顧客希望查詢或刪除個人資料，請直接向店家提出請求（聯絡方式見各店公告）。

店家可視自身需求，在後台「訂單明細」頁面匯出或刪除訂單。

---

### 使用的第三方服務

| 服務 | 用途 | 服務提供方 |
|------|------|-----------|
| Firebase Authentication | 後台帳號驗證與登入 | Google LLC |
| Cloud Firestore | 訂單與菜單資料儲存 | Google LLC |
| Firebase Hosting | 網頁部署與提供 | Google LLC |
| api.qrserver.com | 產生 QR Code 圖片 | goQR.me |

上述第三方服務各有其獨立的隱私權政策，本系統無法控制其資料處理方式。

---

### 使用者權利

依個人資料保護法，您對於本人的個人資料享有以下權利：

- **查詢**：向店家請求確認是否持有您的個人資料
- **閱覽**：請求取得您的個人資料副本
- **更正**：請求更正不正確的個人資料
- **刪除**：請求刪除不再需要保存的個人資料
- **停止蒐集**：請求停止蒐集或使用您的個人資料

如需行使上述權利，請透過店家公告的聯絡方式提出申請。

## 二、服務條款草稿

### 服務說明

本系統提供線上點餐服務，讓顧客透過掃描 QR Code 或直接開啟網址進行點餐，以及讓店家透過後台管理訂單、菜單與帳號。

---

### 菜單與價格

- 菜單品項、價格、供應狀態及營業時間，以店家於後台公告的資訊為準
- 店家保留隨時調整菜單、價格及暫停供應特定品項的權利
- 若品項於訂單送出後缺貨，店家將盡快以電話或現場方式通知顧客

---

### 訂單處理

- 點擊「送出訂單」後，訂單資料將傳送至店家後台
- **訂單送出不等於付款完成**，付款方式依各店家規定（現金、電子支付等）於現場進行
- 店家保留接受、取消或調整訂單的權利（例如：品項售完、訂單資訊有誤）
- 訂單一旦送出，如需修改或取消，請立即向店家現場說明

---

### 使用者責任

- 使用者應提供正確的姓名、電話等聯絡資料，以確保店家能順利聯繫
- 提供不實資訊導致無法出餐或聯絡，由使用者自行負責

---

### 服務中斷與免責

- 本系統透過 Google Firebase 提供服務，可能因網路狀況、Firebase 服務維護、使用裝置等因素導致短暫中斷
- 本系統不保證服務完全不中斷或不發生錯誤
- 因不可抗力（天災、系統停機、網路中斷等）導致訂單未傳送，店家與系統開發方不負賠償責任

---

### 條款修改

店家保留修改本服務條款的權利。條款更新後，繼續使用本系統即視為同意新條款。

## 四、後續實作建議

目前版本尚未在前台加入法律文件連結，正式上線前可視需要補充：

### 最低建議（對顧客透明）

在前台頁尾（`index.html` 的 `<footer>`）或訂單確認頁加入小字連結：

```html
<p>送出訂單即表示您同意我們的
  <a href="/privacy" target="_blank">隱私權政策</a> 與
  <a href="/terms" target="_blank">服務條款</a>
</p>
```

### 較嚴謹做法（明確同意）

在「送出訂單」按鈕上方加入勾選框：

```html
<label>
  <input type="checkbox" id="agreeTerms" required />
  我已閱讀並同意
  <a href="/privacy" target="_blank">隱私權政策</a> 與
  <a href="/terms" target="_blank">服務條款</a>
</label>
```

送出前在 JS 驗證 `document.getElementById("agreeTerms").checked`。

### 文件放置方式

可以選擇：
1. 在 Firebase Hosting 新增 `/privacy.html` 與 `/terms.html` 頁面
2. 或將本草稿整理後以 Google 文件對外分享連結
3. 或放在店家官網的對應頁面

> 實作時機：確認草稿內容後，列為 P0 的一部分，於正式上線前完成。
