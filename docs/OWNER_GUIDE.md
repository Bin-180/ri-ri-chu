# 老闆操作手冊

本手冊說明老闆（owner）帳號在後台能做的所有操作。

---

## 登入後台

- 網址：`https://ririchu-14b81.web.app/admin.html`
- 輸入 owner 帳號的 Email 與密碼，按「登入」
- 登入後會直接進入「收單」頁面

> 若忘記密碼，請聯絡系統建置人員重設。目前系統無自助改密碼功能。

---

## 各功能說明

### 收單（即時訂單）

登入後預設顯示此頁面，會自動更新，不需手動重新整理。

- 每筆訂單會顯示：桌號或姓名、內用／外帶、品項明細、總金額、備註、下單時間
- **完成出餐**：點訂單右下角「✓ 完成出餐」按鈕，該訂單會從收單畫面消失，並移入訂單明細
- 上方橘色數字標示目前待處理訂單數
- 若同時有多筆訂單，從上往下依序處理即可

---

### 訂單明細

查看已完成的歷史訂單。

- 可切換「日」或「月」檢視
- 點日期欄位可選擇要查詢的日期或月份
- 每筆訂單顯示：時間、桌號/姓名、品項、總金額

#### 匯出 CSV

- 在「訂單明細」頁面右上角，按「匯出 CSV」
- 系統會依目前選擇的日期範圍，將所有訂單匯出成 `.csv` 檔案
- 檔案名稱格式：`orders-日期.csv`（例如 `orders-2026-05-24.csv`）
- 以 Excel 開啟時中文顯示正常（含 UTF-8 BOM）
- 欄位包含：訂單ID、時間、服務方式、桌號/姓名、電話、品項、備註、金額、狀態

---

### 統計數據

查看營業金額與訂單量。

- 可切換「日」或「月」檢視
- 顯示：訂單數、營業額、平均客單價
- 下方顯示各品項銷售排行

---

### 菜單管理

新增、修改、下架品項。

- 按「＋ 新增品項」填寫品項名稱、分類、價格，按儲存
- 每個品項旁有「編輯」（鉛筆圖示）與「下架」開關
  - 下架後品項仍保留，前台不顯示；重新開啟即可恢復
- 若品項有多種選擇（例如大杯／小杯），可在「規格」欄位新增

#### 菜單排序

- 按右上角「排序」按鈕進入排序模式
- 每個品項旁會出現 ↑ ↓ 按鈕，點擊調整順序
- 左側可調整**分類的排列順序**（整個分類區塊上下移動）
- 調整完畢後按「完成排序」儲存，按「取消」放棄變更

---

### 店家設定

修改店家基本資料，會即時反映在前台（顧客點餐頁）與 QR Code 頁。

| 欄位 | 說明 |
|------|------|
| 店名 | 顯示在前台頁首與 QR Code 名片 |
| 副標題 | 顯示在店名下方，例如「早餐・早午餐」 |
| 地址 | 顯示在 QR Code 名片 |
| 電話 | 顯示在 QR Code 名片 |
| 營業時間 | 顯示在 QR Code 名片，例如「06:00 – 13:30」 |
| 公休說明 | 顯示在 QR Code 名片，例如「週四公休」 |
| 個資聯絡 Email | 顯示於「隱私權政策與服務條款」頁面，供顧客申請個資查詢或刪除時聯絡使用（選填） |
| Logo 網址 | 見下方說明 |

修改完後按「儲存設定」，頁面右上角會出現「已儲存」提示。

#### 設定 Logo

本系統 Logo 使用**網址連結**（非直接上傳圖片）。

步驟：
1. 將 Logo 圖片上傳到任何可公開存取的圖床（例如 Imgur、Google 相簿公開連結、Cloudinary 等）
2. 複製圖片的直接連結（網址結尾通常是 `.jpg` 或 `.png`）
3. 貼入「Logo 網址」欄位，按「儲存設定」
4. 前台頁首與 QR Code 名片即會更新

> 若暫時沒有 Logo，前台會使用預設圖片，不影響點餐功能。

---

### QR Code

產生與列印供顧客掃描的 QR Code 名片。

- 切換到「QR Code」tab，系統自動產生包含店名、地址、電話的名片樣式
- QR Code 掃描後會直接開啟本店的點餐頁面
- 按「列印 QR Code」可開啟列印視窗，直接印出

> QR Code 指向的網址是 `https://ririchu-14b81.web.app/?store=ririchu`

---

### 員工帳號

新增或停用員工（staff）與管理員（manager）帳號。

#### 新增帳號

1. 切換到「員工帳號」tab
2. 填入顯示名稱、Email、密碼，選擇角色（staff 或 manager）
3. 按「新增帳號」

角色差異：
- **staff**：只能看到「收單」，可接單完成出餐
- **manager**：可看「收單、訂單明細、統計數據、QR Code」

#### 停用帳號

- 員工帳號列表中，每筆旁有「停用」開關
- 停用後該帳號無法再登入，但資料保留

> 停用的帳號仍可重新啟用。帳號無法刪除（Firestore 安全規則限制）。

---

### 系統檢查

快速確認系統各項設定是否正常。

- 切換到「系統檢查」tab，系統自動執行以下檢查：
  - 店家設定（店名、電話、營業時間、Logo、主題色彩）
  - 菜單（品項數量、供應狀態、分類、排序）
  - 帳號（owner 帳號確認、員工帳號是否存在）
  - QR Code / PWA（網址設定、manifest、service worker）
- 結果分三種：
  - ✅ 通過：正常
  - ⚠️ 警告：建議處理，但不影響點餐
  - ❌ 錯誤：需要立即處理
- 若出現警告或錯誤，項目名稱可點擊，系統會直接跳到對應設定頁
- 按「重新檢查」可重新執行
